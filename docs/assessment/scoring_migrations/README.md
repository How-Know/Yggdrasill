# 점수 마이그레이션 로그

이 디렉터리는 점수 체계 변경 이력을 시간순으로 누적 관리한다.

## 파일명 규칙

- `YYYYMMDD_NNN_<slug>.md`
- 예: `20260212_001_attendance_score_v1.md`

## 운영 규칙

- 점수 관련 코드 변경은 가능한 한 **마이그레이션 문서와 같은 PR/커밋**에 포함한다.
- 한 문서에는 한 개의 명확한 변경 단위만 담는다.
- 변경 이유(why)와 검증 결과를 반드시 남긴다.

## 권장 순서

1. 문서 초안 작성
2. 코드 반영
3. 검증 결과 업데이트
4. 상태를 `draft` -> `applied`로 전환
