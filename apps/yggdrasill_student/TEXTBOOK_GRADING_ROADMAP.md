# 교재 풀기 채점 로드맵

2026-07-09 기준. 정답 DB 전수 감사 결과(5,206문항)와 함께 정리.

## 현재 구현 (v2)

| 분류 | 문항 수 | 채점 방식 |
|---|---|---|
| 객관식 | 1,792 | 자동 (원문자/숫자 정규화, 복수답 순서 무관) |
| 주관식 — 수식/숫자/단위 | 4,780 중 대부분 | 자동 (Edge Function `student_textbook_grade`) |
| 세트형 `(1)(2)` 소문항 | 145 | **셀프 채점 폴백** |
| 복수 라벨 답 (`a=…, b=…`, `x절편: …`) | 178 | **셀프 채점 폴백** |
| `(가)(나)` 빈칸 채우기 | 27 | **셀프 채점 폴백** |
| 행렬/연립 (`\begin{…}`) | 27 | **셀프 채점 폴백** |
| 그림 답 (`answer_kind = image`) | 46 | **셀프 채점 폴백** |
| 풀이 참조 (`풀이 N쪽`) | 3 | **셀프 채점 폴백** |

자동 채점 파이프라인 (`supabase/functions/student_textbook_grade/grading.ts`):

1. LaTeX/선형 표기 정규화 → 문자열 일치
2. `,`/`또는` 목록 → 원소별 다중집합 매칭 (순서 무관)
3. 수치 평가 동치 (분수↔소수, 2^3↔8, 2√3↔√12, ± 전개, 변수식 샘플링)
4. 부등식 구조 비교 (거울 표기 `1<x`↔`x>1` 포함)
5. 단위 해석:
   - 숫자만 맞으면 정답 + `unit_hint` ("단위도 써 보세요")
   - 단위 환산 동치(10m↔1000cm) → 정답. 발문이 단위를 지정했는지 AI가
     판정(문항 본문 `pb_questions.stem` 우선, 없으면 크롭 이미지)해
     지정된 경우 `unit_caution` 표시. AI 실패 시에도 정답 + 주의(안전 기본값)
   - 단위만 다르고 숫자 같음(10m↔10cm) → 정답 + `unit_caution`
6. 한글 서술 답 → 압축 문자열 일치, 불일치 시 AI 동치 판정
   (조사/어순 차이 흡수 — "해는 없다"↔"해가 없다")
7. AI 판정은 `student_grading_ai_cache`에 캐시 (문항×답 단위, 재호출 없음)

- AI 제공자: `GEMINI_API_KEY`(우선) → `OPENAI_API_KEY`(현재 등록됨).
  게이트웨이에서 쓰는 Gemini 키를 `supabase secrets set GEMINI_API_KEY=…`로
  등록하면 Gemini로 전환된다.
- self/auto 분류는 SQL `_student_grading_mode`와 TS `gradingMode()` 양쪽에
  동일 로직 유지 (변경 시 함께 수정).

## 추후 작업

### 1. 세트형(종속형 세트형) 소문항 분리 — 145문항

- **DB**: `textbook_problem_answers`에 소문항별 정답 분리 저장.
  안 1) `answer_parts jsonb` 컬럼 추가: `[{"no":"(1)","text":"…"}, …]`
  안 2) 마이그레이션 도구(학습앱 교재 파트)에서 세트 소문항을 별도 crop처럼
  등록. — 정답 원문이 이미 `(1) … (2) …` 형태라 파서로 1차 자동 분리 가능.
- **UI**: 문항 행을 확장해 소문항별 입력칸 제공, 부분 정답 표시.
- **채점**: 소문항별로 기존 자동 채점 파이프라인 재사용.

### 2. 복수 라벨 답 (a, b 값 / 여러 종류 답) — 178문항

- 입력칸은 **하나만** 제공 (문제를 끝까지 읽는지 평가 목적).
- 파서(규칙 + AI 폴백)로 학생 답에서 라벨별 값 추출 → 라벨별 채점.
- 누락 라벨은 `missing_parts` 플래그로 기록 → "빠뜨린 답이 있어요" 힌트만
  제공(정답 구조는 노출하지 않음). 문항별 누락률을 독해력 지표로 축적.
- DB: `student_textbook_answer_records.flags`에 `missing_parts` 추가 +
  상세는 `result_detail jsonb` 컬럼 신설.

### 3. 그림 답 — 46문항

- 마이그레이션 단계에서 답지 그림을 AI로 구조화(수직선 범위, 표 선택지 등)
  → 구조화 가능한 유형은 전용 입력 UI(수직선 드래그, 표 체크)로 자동 채점.
- 구조화 불가능한 답은 셀프 채점 유지하되, 답지 크롭 이미지를 저장해
  reveal 시 이미지로 노출 (현재는 렌더 PNG/텍스트만).

### 4. 행렬/연립 입력기 — 27문항

- 수식 에디터에 행렬(격자 슬롯)/연립(중괄호 2줄) 구조 원자 추가.
- 직렬화는 pmatrix/cases 호환 선형 표기로.

### 5. 순환소수 UX

- 키패드 `0.ẋ` 키로 직전 숫자에 순환점 부착·토글(구현됨).
- 에디터 미리보기는 결합 문자 대신 점을 직접 그려 폰트 이질감 없음(구현됨).

### 6. 상위 학년 기호 (해당 교재 마이그레이션 시점에 확장)

현재 마이그레이션된 정답 5,206건에는 아래 기호가 등장하지 않는다.
새 교재를 마이그레이션할 때 정답 DB를 다시 감사(`corpus_audit.ts`)하고
필요한 것부터 순서대로 추가한다.

| 기호 | 에디터 | 엔진 |
|---|---|---|
| 선분/호 기호 (AB 위 bar) | overline 구조 원자 필요 | 문자열 비교로 충분 |
| log, sin/cos/tan | 함수 토큰 키 | 평가기에 함수 지원 추가 |
| 극한 lim, 시그마 ∑, 적분 ∫ | 위·아래 첨자 구조 원자 | 평가 불가 → 구조 비교 + AI 동치 |
| 도함수 f'(x), dy/dx | ' 키 + 분수 재사용 | 문자열/구조 비교 |
| 행렬/연립 | 격자·중괄호 구조 원자 (§4) | 원소별 비교 |
| 아래첨자 수열 a_n | `_()` 구조 원자 | 문자열 비교 |

- 기하 기호(∠ △ □ ≡ ∽ ⊥ ∥ ⌒ ° ○)와 단위 선반(길이/넓이/부피/질량/시간
  등)은 키패드에 **이미 추가됨** — 중등 기하 교재 마이그레이션 대비.
- ⁿ√(세제곱근·네제곱근)은 에디터·엔진 모두 **구현됨** (`√[n](x)` 표기,
  음수 홀수근 지원).

## 알려진 한계

- 발문 단위 지정 AI 판정은 `pb_question_uid` 연결(현재 0건)이 채워지면
  본문 기반으로 정확도가 오른다. 그 전까지는 크롭 이미지 판독.
- ML Kit 필기 인식(`en` 모델)은 숫자/영문/기본 기호 위주 — √, ≤ 같은 특수
  기호는 수식 에디터 사용 권장. iOS 시뮬레이터/macOS에서는 미지원(실기기 전용).
